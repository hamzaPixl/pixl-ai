# Humanizer

## Goal

Natural, human-sounding writing by avoiding AI tells.

## Words to Avoid

### High Priority

| Avoid | Use Instead |
|-------|-------------|
| delve | explore, examine, look at |
| leverage | use, apply |
| utilize | use |
| facilitate | help, enable, allow |
| endeavor | try, attempt |
| comprehensive | complete, full, thorough |

### Medium Priority

| Avoid | Use Instead |
|-------|-------------|
| streamline | simplify, improve |
| robust | strong, reliable |
| cutting-edge | modern, new, advanced |
| innovative | new, creative |
| seamless | smooth, easy |
| optimal | best, ideal |

## Phrases to Avoid

| Avoid | Use Instead |
|-------|-------------|
| dive into | explore, examine |
| key takeaway | main point, summary |
| it is important to note | (just state the fact) |
| in conclusion | (often unnecessary) |
| furthermore | also, and |
| consequently | so, as a result |
| aforementioned | (use specific reference) |

## Structural Tells

Avoid these patterns:
- "Let's explore..." openers
- "In this article..." meta-references
- "Firstly... Secondly... Thirdly..." enumeration
- "World of..." filler
- "Game-changer" hype
- "Embrace" as a verb for adoption

## Revision Process

1. **Write first** — get the content down without self-editing
2. **Read aloud** — if it sounds unnatural spoken, rewrite it
3. **Check patterns** — scan for words/phrases from the tables above
4. **Simplify** — replace fancy words with simple ones, cut unnecessary words, lead with the main point
5. **Add specifics** — replace vague claims with numbers, names, and examples

## Quick Checklist

- [ ] No words from the "avoid" tables?
- [ ] No AI structural patterns?
- [ ] Sentences vary in length?
- [ ] Active voice preferred over passive?
- [ ] Specific examples instead of vague claims?
- [ ] Would a human actually say this?
- [ ] Main point comes first, not buried?

## When It Matters Most

- PR descriptions
- Documentation and READMEs
- User-facing content
- Commit messages
- Blog posts and articles

## When It Matters Less

- Code comments (clarity over style)
- Log messages
- Temporary notes
